SMTP Server Per User


Features

  • Allows every user to send mail with their own separate outgoing mail server

  • Replaces relay email adderss eg: <bounce+58-crm.lead-24@ domain.com> to sender's address which is not blocked by mail servers.

  • Fixes email routing. First it checks the parent messages of incoming email, if not found then falls back on the alias.  unlike odoo standard first it falls back on the alias ignoring the parent message

  • On failing authentication on Incoming/Outgoing mail server. It creates activity to configure incoming outgoing server from a single form

Configure Odoo to use outgoing email server by user.


Create a regular user, don't forget to set email alias equal to email server.

Outgoing Mail Server

Create outgoing mail server same as user alias. While sending an email it will auto-detect correct outgoing mail server. If none of them matches then it would use the 1st one found with the lowest priority

Mail Server Configuration

On failing Incoming/Outgoing mail. User gets notification to update his own password for incoming/outgoing mail server
